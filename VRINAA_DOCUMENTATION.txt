1. PROJECT OVERVIEW
================================================================================

Vrinaa Occasions is a event decoration services website built with a modern
full-stack architecture. The site showcases event decoration services,
allows users to view galleries, event types, and contact the business via
a contact form with EmailJS integration.

Key Features:
- Event decoration service showcase
- Photo gallery with event images
- Contact form with email notifications (EmailJS)
- Responsive design for all devices
- Multiple pages: Home, About, Events, Gallery, Contact, Terms

CONTACT FORM (EmailJS Integration)
----------------
Flow:
1. User fills form (name, email, phone, eventType, eventDate, message)
2. User clicks "Submit Inquiry"
3. React captures form data via useState
4. EmailJS SDK sends data to EmailJS service
5. EmailJS forwards email to configured recipient
6. Success/error toast notification shown

----------------
FASTAPI APPLICATION STRUCTURE
----------------
server.py

1. Imports
   - FastAPI, APIRouter (web framework)
   - Motor (MongoDB async driver)
   - Pydantic (data validation)
   - dotenv (environment variables)
   - CORS middleware

2. Configuration
   - Load .env file
   - MongoDB connection via MONGO_URL
   - Database name from DB_NAME
   - CORS origins from CORS_ORIGINS

3. Models (Pydantic)
   - StatusCheck: Data model for status check records
     * id: UUID string
     * client_name: string
     * timestamp: datetime

4. Routes (APIRouter with /api prefix)
   - GET  /api/           : Root endpoint - returns {"message": "Hello World"}
   - POST /api/status     : Create a new status check
   - GET /api/status      : Get all status checks

5. Middleware
   - CORS: Allows requests from http://localhost:3000

6. Events
   - on shutdown: Close MongoDB connection

----------------
RUNNING THE BACKEND
----------------
Development:
   cd backend
   python server.py
   # OR
   uvicorn server:app --host 0.0.0.0 --port 8000 --reload

Production:
   uvicorn server:app --host 0.0.0.0 --port 8000

7. API ENDPOINTS
================================================================================

Base URL: http://localhost:8000/api

----------------
ENDPOINTS
----------------

1. GET /
   URL: http://localhost:8000/api/
   Description: Root endpoint for health check
   Response: {"message": "Hello World"}

2. POST /status
   URL: http://localhost:8000/api/status
   Description: Create a new status check record
   Request Body:
   {
     "client_name": "string"
   }
   Response:
   {
     "id": "uuid-string",
     "client_name": "string",
     "timestamp": "datetime"
   }

3. GET /status
   URL: http://localhost:8000/api/status
   Description: Get all status check records
   Response: Array of status check objects
   [
     {
       "id": "uuid-string",
       "client_name": "string",
       "timestamp": "datetime"
     }
   ]

9. DEPLOYMENT
================================================================================

----------------
LOCAL DEVELOPMENT
----------------
Prerequisites:
- Node.js v18+
- Python v3.10+
- MongoDB (local or Atlas)

Frontend:
   cd frontend
   npm install
   npm start
   # Runs on http://localhost:3000

Backend:
   cd backend
   python -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   python server.py
   # Runs on http://localhost:8000

----------------
VERCEL DEPLOYMENT (Frontend)
----------------
1. Push code to GitHub
2. Import project in Vercel
3. Configure:
   - Framework: Create React App
   - Build Command: npm run build
   - Output Directory: build
4. Add Environment Variables:
   - REACT_APP_EMAILJS_SERVICE_ID
   - REACT_APP_EMAILJS_TEMPLATE_ID
   - REACT_APP_EMAILJS_PUBLIC_KEY
5. Deploy

----------------
ENVIRONMENT VARIABLES
----------------
Backend (.env):
   MONGO_URL=mongodb+srv://...
   DB_NAME=vrinaa_occasions
   CORS_ORIGINS=http://localhost:3000

Frontend (.env.local):
   REACT_APP_EMAILJS_SERVICE_ID=service_heuxk2k
   REACT_APP_EMAILJS_TEMPLATE_ID=template_7s1eg5k
   REACT_APP_EMAILJS_PUBLIC_KEY=ZP1l01ijVcijrBEfF
